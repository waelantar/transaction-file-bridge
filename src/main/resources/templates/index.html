Project Overview

This microservice monitors a local directory for banking transaction CSV files, validates and transforms them into JSON format, and securely uploads them to MinIO storage using Apache Camel integration. The application is containerized with Docker and deployed to Red Hat OpenShift Sandbox with a comprehensive CI/CD pipeline integrated with GitHub Actions.

Built with robust error handling mechanisms including retry logic and dead-letter queue patterns, the system ensures data integrity and reliability. Monitoring capabilities are provided through Spring Boot Actuator endpoints, and the user interface is implemented with Thymeleaf templates for a seamless experience.

Key Features

File Monitoring

Automatically detects new CSV files in the designated input directory for processing.

Data Transformation

Converts banking transaction data from CSV format to structured JSON for downstream systems.

Secure Storage

Uploads processed files to MinIO object storage with secure authentication mechanisms.

Error Handling

Implements retry logic and dead-letter queue patterns for robust error management.

Health Monitoring

Comprehensive health checks and metrics through Spring Boot Actuator endpoints.

CI/CD Pipeline

Automated deployment pipeline integrated with GitHub Actions for continuous delivery.
